## Key Concepts

### Class

A blueprint used to create objects.

### Object

An instance of a class containing actual data.

### Properties

Used to define and access class data using `get` and `set`.

### Encapsulation

Protects data by controlling access through properties and methods.

### Inheritance

Allows a child class to acquire properties and methods from a parent class.

### Polymorphism

Allows the same method to behave differently based on the object calling it.

### Abstraction

Hides implementation details and exposes only essential functionality using abstract classes and methods.

### Interface

Defines a contract that implementing classes must follow.
